# PatchTrap — Investor-Ready Design Spec
**Date:** 2026-04-27
**Status:** Approved

---

## 1. Goal

Transform the current single-file prototype (`patchtrap_mil.py`) into a production-quality, pip-installable Python security package ready to demo to investors and publish on PyPI.

Deliverables:
- `pip install patchtrap` works, CLI command is `patchtrap`
- Full pytest suite with green CI badge
- Standalone HTML report viewer (no server, opens in browser)
- Attack example scripts + one-command investor demo
- `standalone/patchtrap_mil.py` preserved as the iconic single-file artifact

---

## 2. Architecture

### 2.1 Package structure

```
patchtrap/
├── __init__.py          # Public API: PatchTrapMIL, Policy, SecurityError
├── core.py              # PatchTrapMIL, Policy, SecurityError
├── fingerprint.py       # _fingerprint_callable, _resolve_dotted
├── crypto.py            # SHA256 utils, Merkle tree, chain, HMAC/Ed25519 signing
├── report.py            # report_load, report_verify_integrity
├── viewer.py            # generate_html_report() -> self-contained HTML string
└── cli.py               # argparse CLI, main() entry point

standalone/
└── patchtrap_mil.py     # Original single-file, kept as reference artifact

examples/
├── evil_open.py         # Monkey-patches builtins.open (file spy)
├── evil_socket.py       # Monkey-patches socket.socket (network redirect)
├── evil_meta_path.py    # Injects malicious importer into sys.meta_path
├── evil_env.py          # Exfiltrates via os.environ mutation
└── run_demo.py          # One-command investor demo: runs all attacks, opens HTML

tests/
├── test_crypto.py
├── test_fingerprint.py
├── test_core.py
├── test_report.py
├── test_signing.py
├── test_viewer.py
└── test_cli.py

pyproject.toml
requirements-dev.txt
README.md                # Updated: badges, pip install, demo section
```

### 2.2 Module responsibilities

| Module | Responsibility |
|---|---|
| `core.py` | PatchTrapMIL class: seal, scan, run, event ingestion. Policy, SecurityError. |
| `fingerprint.py` | Bytecode-based fingerprinting for callables, builtins, arbitrary callables. Dotted name resolver. |
| `crypto.py` | SHA256 helpers, canonical JSON encoder, Merkle tree, hash chain, HMAC sign/verify, Ed25519 sign/verify. |
| `report.py` | Load JSON report, rebuild Merkle + chain to verify integrity. |
| `viewer.py` | generate_html_report(report: dict) -> str — self-contained HTML with inline CSS/JS. |
| `cli.py` | patchtrap run, verify, sign, verify-sign, report subcommands. |

### 2.3 Data flow

```
patchtrap run target.py
    -> cli.py parses args
    -> core.PatchTrapMIL(targets, options)
    -> .seal()        [fingerprint.py: fingerprint each target]
    -> .run()         [runpy.run_path + pre/during/post scan]
        -> _scan_once() [fingerprint.py: re-fingerprint, compare]
            -> tamper detected -> _ingest(event) [crypto.py: leaf_hash, chain_next]
            -> auto_restore -> setattr(container, attr, original)
    -> report dict built [crypto.py: merkle_root, chain_tip]
    -> canonical JSON written to patchtrap_report.json
    -> exit code returned

patchtrap report --file report.json
    -> report.py: load + verify integrity
    -> viewer.py: generate_html_report(report)
    -> write report.html, open in browser
```

## 3. CLI

### Commands (unchanged behavior, new entry point)

```bash
patchtrap run target.py --watch builtins.open,socket.socket --auto-restore 1
patchtrap verify patchtrap_report.json
patchtrap sign --file patchtrap_report.json --hmac-key "secret"
patchtrap verify-sign --file patchtrap_report.json --hmac-key "secret"
patchtrap report --file patchtrap_report.json   # NEW: generate + open HTML
```

## 4. HTML Viewer

Generated by viewer.py. Self-contained: no CDN, no external assets.

### Color coding

| Event kind | Color |
|---|---|
| `replaced` | Red |
| `restore` | Green |
| `env_changed` | Orange |
| `meta_path_changed` | Purple |
| `patchtrap.start` / `system_exit` | Gray |

### Features
- Dark mode, sober style
- Status badge: green CLEAN (0 alerts) or red TAMPER DETECTED (alerts > 0)
- Merkle root + chain tip with verified marker if report_verify_integrity passes
- Collapsible raw JSON section
- `patchtrap report` opens the HTML in the default browser via webbrowser.open

## 5. Tests

| File | What is tested |
|---|---|
| `test_crypto.py` | Merkle (empty/single/even/odd leaves), chain tip reproducibility, canonical_bytes stability |
| `test_fingerprint.py` | Fingerprint Python function unchanged/changed, builtin, arbitrary callable, _resolve_dotted |
| `test_core.py` | seal(), detect replacement, auto-restore, fail-closed SecurityError, env change alert, meta_path change alert, clean run = 0 alerts |
| `test_report.py` | Valid report verifies OK, falsified alert -> integrity check fails |
| `test_signing.py` | HMAC sign + verify valid, tampered file -> invalid |
| `test_viewer.py` | generate_html_report returns HTML containing Merkle root, alert count, event kinds |
| `test_cli.py` | patchtrap run exits 0 and creates report, patchtrap verify exits 0, patchtrap --help exits 0 |

All tests use only stdlib. test_cli.py uses subprocess to invoke the installed CLI.

## 6. Examples and Investor Demo

### Attack scripts (examples/)

- `evil_open.py`: replaces builtins.open with a spy that logs every file path accessed
- `evil_socket.py`: replaces socket.socket to redirect connections
- `evil_meta_path.py`: prepends a malicious importer to sys.meta_path
- `evil_env.py`: sets SECRET_KEY=exfiltrated in os.environ

### One-command demo

```bash
python examples/run_demo.py
```

Runs all 4 attack scripts under PatchTrap in sequence, then generates and opens the HTML report in the browser.
